01

什么是 CLI？

CLI（Command Line Interface，命令行界面）是用户与 Agent 交互的文字界面。它就像终端里的 Claude Code——你输入问题或指令，Agent 理解后执行并返回结果。

🍽️

餐厅点餐比喻

你（用户）向服务员（CLI）点菜，服务员传达给厨房（Agent），厨房做好后服务员把菜端给你。

🚗

汽车仪表盘比喻

CLI 就像汽车仪表盘——显示状态信息，接受你的操作指令（加速、刹车、转向），然后控制车辆执行。

🔊

智能音箱比喻

智能音箱是语音界面，CLI 是文字界面——都是连接你和 AI 的桥梁，只是交互方式不同。

为什么选择 CLI？

• 高效：键盘输入比鼠标点击更快
• 可脚本化：可以批量自动化操作
• 资源少：不需要图形界面，服务器也能运行
• 开发者友好：程序员天然习惯终端操作

AgentForgeCLI 核心类

整个 CLI 由一个核心类驱动，它包含 6 个关键方法：

cli.py 
class AgentForgeCLI:
    """AgentForge 命令行界面"""

    def __init__(self):
        # 初始化：加载配置、创建 Agent 实例
        self.config = load_config()
        self.agent = Agent(self.config)
        self.history = []
        self.stats = {"turns": 0, "tools": 0}

    def print_banner(self):
        # 打印 ASCII 艺术横幅
        print(BANNER_ART)

    def print_help(self):
        # 显示所有可用命令的帮助信息
        print(HELP_TABLE)

    def run_agent(self, user_input):
        # 将用户输入发送给 Agent，流式输出结果
        response = self.agent.chat(user_input)
        return response

    def process_command(self, cmd):
        # 解析并执行 / 开头的内置命令
        if cmd == "/help":  self.print_help()
        elif cmd == "/exit": self.exit()
        # ... 其他命令

    def run(self):
        # 主入口：启动横幅 → 主循环
        self.print_banner()
        while True:
            user_input = input("🤖 AgentForge › ")
            if user_input.startswith("/"):
                self.process_command(user_input)
            else:
                self.run_agent(user_input)

类设计思路

• 单一职责：CLI 只负责交互，Agent 只负责推理
• 状态管理：history 记录对话，stats 记录统计
• 命令分发：process_command 根据前缀路由到具体方法
• 主循环：run() 方法是程序的心脏，不断"读取-处理-输出"

def run(self):
    # Step 1: 打印横幅
    self.print_banner()

    # Step 2: 检查 API Key
    if not self.config.get("api_key"):
        print(Colors.RED + "❌ 未找到 API Key！")
        print("请设置环境变量 OPENAI_API_KEY")
        print("或在 .env 文件中配置")
        return

    # Step 3: 主循环
    print(Colors.GREEN + "✅ 就绪！输入问题开始对话\n")
    while True:
        try:
            user_input = input("🤖 AgentForge › ").strip()
            if not user_input: continue
            if user_input.startswith("/"):
                self.process_command(user_input)
            else:
                self.run_agent(user_input)
        except KeyboardInterrupt:
            print("\n👋 再见！")
            break

config.py 
import os
from dotenv import load_dotenv

def load_config():
    """加载配置，按优先级：env > .env > defaults"""
    load_dotenv()  # 加载 .env 文件

    config = {
        # API Key: 优先从环境变量读取
        "api_key": os.getenv("OPENAI_API_KEY", ""),

        # 模型: 环境变量 > 默认值
        "model": os.getenv("MODEL", "gpt-4"),

        # 温度参数: 环境变量 > 默认值
        "temperature": float(
            os.getenv("TEMPERATURE", "0.7")
        ),
    }

    return config

🎨 深入一：ANSI 颜色代码

终端中的颜色通过 ANSI 转义序列 实现。格式为 \033[XXm，其中 XX 是颜色代码。

class Colors:
    """ANSI 颜色代码"""
    RESET   = "\033[0m"    # 重置所有样式
    RED     = "\033[91m"   # 红色 - 错误
    GREEN   = "\033[92m"   # 绿色 - 成功
    YELLOW  = "\033[93m"   # 黄色 - 警告
    BLUE    = "\033[94m"   # 蓝色 - 信息
    MAGENTA = "\033[95m"   # 紫色
    CYAN    = "\033[96m"   # 青色
    BOLD    = "\033[1m"    # 加粗

# 使用示例
print(Colors.GREEN + "✅ 成功！" + Colors.RESET)
print(Colors.RED + "❌ 出错了！" + Colors.RESET)

🛡️ 深入二：输入验证与安全

用户输入是不可信的，CLI 需要做严格的验证：

def process_command(self, cmd):
    """安全地处理用户命令"""
    cmd = cmd.strip().lower()

    # 1. 空命令检查
    if not cmd:
        return

    # 2. 命令白名单（只允许已知命令）
    valid_commands = {
        "/help", "/exit", "/clear",
        "/status", "/history", "/reset",
        "/model", "/config", "/tools", "/exec"
    }
    base_cmd = cmd.split()[0]

    if base_cmd not in valid_commands:
        print(f"❌ 未知命令: {base_cmd}")
        print("输入 /help 查看可用命令")
        return

    # 3. 参数长度限制
    if len(cmd) > 1000:

🔧 深入三：工具调用可视化

def display_tool_call(tool_name, params, result):
    """可视化工具调用"""

    # 1. 显示调用（橙色）
    print(Colors.YELLOW +
          f"🔧 {tool_name}({json.dumps(params)})" +
          Colors.RESET)

    # 2. 显示结果（绿色/红色）
    if result["success"]:
        print(Colors.GREEN +
              f"  → ✅ {result['summary']}" +
              Colors.RESET)
    else:
        print(Colors.RED +
              f"  → ❌ {result['error']}" +
              Colors.RESET)

    # 3. 更新统计
    self.stats["tools"] += 1

这种可视化让用户能实时看到 Agent 的行动，而不是只看到最终答案。这建立了信任——你知道 AI 做了什么、用了什么工具、结果如何。

📝 本章总结

🎯 你学到了什么

  • CLI 是 Agent 的交互面孔，用文字对话驱动 AI
  • AgentForgeCLI 类包含 6 个核心方法：init、print_banner、print_help、run_agent、process_command、run
  • ASCII 横幅用 Box Drawing 字符绘制，给程序一个专业的"门面"
  • 命令系统以 / 开头，分基础、配置、工具三大类
  • 启动流程：加载配置 → 打印横幅 → 检查 API Key → 进入主循环
  • 主循环：显示提示 → 获取输入 → 判断类型 → 执行命令或调用 Agent
  • 配置优先级：环境变量 > .env 文件 > 默认值
  • ANSI 转义码让终端输出五彩缤纷
  • 输入验证和白名单是安全的基石
  • 工具调用可视化让 Agent 的行为透明可信

🚀 最佳实践

• 清晰的提示词：告诉 Agent 你想要什么，越具体越好
• 分步执行：复杂任务拆成小步骤，逐步完成
• 善用命令：用 /status 检查状态，用 /history 回顾对话
• 关注工具调用：观察 Agent 用了什么工具，确认方向正确
• 及时重置：话题切换时用 /reset 清理上下文
